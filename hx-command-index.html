Overview

命令总览

当前版本暴露 16 个子命令，数量以 bin/hx.js 的分发映射为准。

命令 分组 主要作用
hx setup安装与维护写入用户级 profiles、Claude 命令和 skills
hx init安装与维护在业务项目中初始化目录、配置和命令模板
hx upgrade安装与维护升级项目内框架痕迹和命令文件
hx uninstall安装与维护清理项目内安装痕迹
hx version安装与维护输出 CLI 版本
hx doc文档与计划创建需求文档
hx plan文档与计划生成计划 Markdown 和 progress JSON
hx ctx文档与计划校验上下文和 Profile 资源完整性
hx done文档与计划回写任务完成状态
hx run执行与复核生成任务级 Agent Prompt
hx review执行与复核输出 review checklist
hx fix执行与复核根据错误日志生成修复 Prompt
hx gate质量与交付执行 Profile 定义的质量门控

hx mr质量与交付汇总 MR 创建上下文
Bootstrap

安装与维护命令

该组命令负责安装、初始化、升级和卸载。

Setup

hx setup

hx setup [--dry-run]

初始化用户级共享资源。

会同步 ~/.hx/profiles/、~/.claude/commands/、~/.claude/skills/。

~/.hx/config.json 不存在时会创建空配置文件。

Init

hx init

hx init [--profile <name>] [--target <dir>]

在业务项目中安装本地工作流骨架。

会创建 .hx/config.json、docs/requirement/、docs/plans/、AGENTS.md、.CLAUDE.md。

也会把框架命令模板和 skills 安装到项目内的 .claude/。

Upgrade

hx upgrade

hx upgrade [--target <dir>] [--dry-run]

升级已初始化项目中的框架痕迹。

会更新命令文件、skills 和 CLAUDE 标记块。

.hx/config.json 配置了 pinnedCommands 时，对应命令不会被覆盖。

Uninstall

hx uninstall

hx uninstall [--target <dir>] [--yes] [--dry-run]

清理项目内由框架安装的目录、命令文件和 CLAUDE 标记块。

--yes 跳过确认提示。

--dry-run 只展示将删除哪些内容，不实际执行。

Version

hx version

hx version

输出当前 CLI 版本号。也支持 hx --version 与 hx -v。

Planning

文档与计划命令

该组命令负责需求、计划与上下文准备。

Doc

hx doc

hx doc <feature-name> [--profile <team[:platform]>] [--task <id>]

从 Profile 对应的需求模板创建需求文档。

feature-name 必须是 kebab-case。

--task 可把 DevOps 任务号写进文档头部。

Plan

hx plan

hx plan <feature-name> [--profile <team[:platform]>]

读取需求文档，生成计划 Markdown 与 *-progress.json。

需求文档不存在时，会提示先运行 hx doc。

生成后会把活跃特性写回 AGENTS.md。

Ctx

hx ctx

hx ctx [--profile <team[:platform]>]

校验 AGENTS.md、需求文档、计划文件和 Profile 资源是否一致。

会检查活跃特性是否缺少需求文档或 AC。

也会检查 progress JSON 是否缺少任务信息或引用了不存在的文档。

Done

hx done

hx done <TASK-ID>

标记指定任务完成，并输出下一个待办任务。

会扫描 docs/plans/ 下所有 progress JSON 寻找对应任务。

全部任务完成后，会提示继续运行 hx entropy。

Execution

执行与复核命令

该组命令负责执行任务与复核结果。

Run

hx run

hx run <feature-name> <TASK-ID> [--profile <team[:platform]>] hx run <TASK-ID> [--profile <team[:platform]>]

读取任务上下文，生成 Agent Prompt。

计划文件中的 profile 与显式参数不一致时，会直接报错退出。

任务已是 done 状态时，不会重复生成执行 Prompt。

Review

hx review

hx review [--profile <team[:platform]>]

输出当前 Profile 的 review checklist。

Profile 配置了 reviewExtra 时，会在清单后追加平台检查项。

Fix

hx fix

hx fix [--profile <team[:platform]>] [--log <text>] [--file <path>]

根据错误日志生成修复 Prompt。

日志优先级是 --file → --log → 自动执行 npm run hx:test 截取尾部输出。

会加载团队黄金规则和 fix Hook。

Quality & Release

质量与交付命令

该组命令负责执行后的校验、清理和交付。

Gate

hx gate

hx gate [--profile <team[:platform]>] [--scheme <name>]

从 Profile 的 gate_commands 中按顺序执行门控。

固定顺序是 lint → build → type → test → arch，只执行已定义步骤。

命令中存在未替换占位符时，会要求补齐 CLI 参数，例如 --scheme。

Check

hx check

hx check [--profile <team[:platform]>]

综合检查入口，顺序执行 hx ctx 和 hx gate。

适用于 CI 或本地自检场景。

Entropy

hx entropy

hx entropy

扫描 src/ 下的熵增模式，如 console.log、: any、裸 Error、TODO 等。

当前实现不区分 profile，扫描对象是项目的 src/**/*.ts(x)，排除测试文件和 .d.ts。

MR

hx mr

hx mr <feature-name> [--project <group/repo>] [--target <branch>]

汇总需求文档、任务完成情况、git log 和 diff stat。

默认目标分支是 main。

可配合 /hx-mr 命令或 GitLab 脚本继续自动化。
