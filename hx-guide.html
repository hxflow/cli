产品定位

HXFlow CLI 是工程输入、团队规则与质量门控的统一入口。

输入治理

结构化工程输入

通过 doc → plan → ctx 固化需求、任务边界和上下文。

团队治理

统一团队差异

通过 Profile 管理团队的架构层级、模板与门控命令。

质量闭环

形成质量闭环

gate、review、fix、entropy 与 mr 负责校验、修复和交付。

i
命名：产品名为 HXFlow，命令为 hx。仓库与 npm 仍保留部分 Harness Workflow 历史命名。
Workflow

核心链路

标准链路为“准备输入 → 执行任务 → 校验与沉淀”。

01
需求与计划
hx doc 创建需求文档，hx plan 生成任务拆分和 progress JSON。
02
上下文校验
hx ctx 先确认 AGENTS.md、需求文档、计划文件和 Profile 资源都处在一致状态。
03
任务执行
hx run 读取任务与上下文，生成执行 Prompt。
04
质量收口
hx gate 执行门控，hx review / hx fix 负责复核与修复。
05
状态回写与交付
hx done 更新任务状态，hx entropy 沉淀问题模式，hx mr 汇总 MR 上下文。

Constraints

关键约束

以下约束决定 CLI 的实际行为。

命令真源

入口和分发以 bin/hx.js 为准

命令数量、帮助信息和分发关系由 CLI 入口定义，具体行为在 src/scripts/hx-*.js 中实现。

bin/hx.js src/scripts/
配置真源

路径、Profile 和 Hook 有固定解析顺序

优先级、默认路径和 Hook 行为由 resolve-context.js、profile-utils.js 与 hooks-utils.js 定义。

resolve-context.js profile-utils.js hooks-utils.js
